## Getting Help

- Use GitHub issues for bugs, broken docs, and concrete feature requests.
- Use repository READMEs for repo-specific setup and local development.
- For security-sensitive issues, follow [SECURITY.md](./SECURITY.md).

When opening an issue, include:

- the repository involved
- the expected behavior
- the observed behavior
- reproduction steps
- logs, screenshots, or request examples if relevant
